把「目前這個專案」這次做的進度，寫進該專案的 RESUME.md 交接檔。我這次想補充/強調的重點是：$ARGUMENTS

## 你要做的事

1. **判斷「目前專案」**：
   - 先看目前工作目錄（用 Bash `pwd`）。若我人在某個專案資料夾裡，就用那個資料夾。
   - 若工作目錄不像專案（例如在家目錄），就問我這次要存哪個專案，或從 `~/Desktop/claude/*/` 讓我選。

2. **找或建 RESUME.md**：
   - 該專案資料夾若已有 `RESUME.md` → 讀進來、在既有內容上更新（不要整個砍掉重寫，保留還有效的資訊）。
   - 若沒有 → 用這個 repo 的 `templates/RESUME.template.md` 當骨架建立一份。

3. **更新內容**：根據「我們這次對話實際做了什麼」＋我的 $ARGUMENTS，更新以下重點，寫得**具體、可執行、含完整路徑**：
   - 一句話現況
   - 這次完成了什麼（含檔案/資料夾路徑、指令）
   - 重要踩雷筆記（之後容易忘的坑）
   - 下次要做什麼（按順序、可勾選）
   - 金鑰/機密的「位置」（只寫位置，**絕不把金鑰內容寫進去**）
   - 更新日期

4. **寫檔**：用 Write/Edit 存回該專案的 `RESUME.md`。

5. **回報**：告訴我存到哪個路徑、這次更新了哪些重點。

## 鐵則
- **永遠不要把任何 API 金鑰、密碼、token 的「內容」寫進 RESUME.md**，只寫它放在哪個檔。
- RESUME.md 的定位是「給明天的自己 / 給另一台電腦的完整交接」，要讓人一看就能無痛接手。
